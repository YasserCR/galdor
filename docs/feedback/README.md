# Feedback

galdor is pre-1.0. The framework covers the surface; the edges only show up
in real deployments. The most valuable thing you can send is a report from
pointing galdor at an actual project.

## Where to send it

- **Found a bug?** Open a [bug report](https://github.com/YasserCR/galdor/issues/new?template=bug_report.md). Include `galdor doctor` output.
- **Want a feature?** Open a [feature request](https://github.com/YasserCR/galdor/issues/new?template=feature_request.md) — an API sketch helps.
- **Built something real?** Open an [integration feedback](https://github.com/YasserCR/galdor/issues/new?template=integration_feedback.md) issue. This is the one we want most.
- **Open question or idea?** Start a [discussion](https://github.com/YasserCR/galdor/discussions).

## What makes integration feedback useful

Specifics about **where it hurt**: friction, missing pieces, confusing
APIs, docs that didn't match reality, and the workarounds you had to write.
That's what shapes the next releases — the first integration retro shaped
most of v0.1.0.

Before filing, run `galdor doctor` and include its output, plus `galdor
version` and which provider(s) you used.
